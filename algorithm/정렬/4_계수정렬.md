# 계수 정렬

```
특정한 조건이 부합할 때만 사용할 수 있지만 매우 빠르게 동작하는 정렬 알고리즘이다.
계수 정렬은 데이터의 크기 범위가 제한되어 정수 형태로 표현할 수 있을때만 사용가능하다.

데이터 개수가 N, 데이터(양수) 증 최댓값이 K일 때 최악의 경우에도 수행 시간 O(N + K)를 보장한다.
```

# step

```
데이터를 하나씩 확인하며 데이터의 값과 동일한 인덱스의 데이터를 1씩 증가시킨다.
(이를 반복한다.)

결과적으로 최종 리스트에는 각 데이터가 몇 번씩 등장했는지 그 횟수가 기록된다.
```

## 방법 1
``` python

# 계수 정렬



arr = [7, 5, 9, 0, 3, 1, 6, 2, 9, 1, 4, 8, 0, 5, 2]
# 모든 원소의 값이 0보다 크거나 같다고 가정

cnt = [0] * (max(arr)+1)
# 모든 범위를 포함하는 리스트 선언(모든 값은 0으로 초기화
# )
for i in range(len(arr)):
    cnt[arr[i]] += 1 # 각 데이터에 해당하는 인덱스 값 증가

# cnt
# [1, 2, 1, 1, 0, 1, 1, 1, 0, 2]

for i in range (len(cnt)): # 리스트에 기록된 정렬 정보 확인
    for _  in range(cnt[i]):
        print(i, end= ' ') # 띄어쓰기를 구분으로 등장한 횟수 만큼 인덱스 출력


```


## 방법 2
``` python

# 계수 정렬

arr = [7, 5, 9, 0, 3, 1, 6, 2, 9, 1, 4, 8, 0, 5, 2]

cnt = [0] * (max(arr)+1)

for i in range(len(arr)):
    cnt[arr[i]] += 1

# cnt
# [1, 2, 1, 1, 0, 1, 1, 1, 0, 2]

ans = []

for i in range (len(cnt)):
    for _  in range(cnt[i]):
        ans.append(str(i))
# ans
# [0, 0, 1, 1, 2, 2, 3, 4, 5, 5, 6, 7, 8, 9, 9]
print(' '.join(ans))

```

# 복잡도 분석

```
계수 정렬의 시간 복잡도와 공간 복잡도는 모두 O(N +K)dlek

1) 극한의 비효율적일 때

계수 정렬은 때에 따라서 심각한 비효율성을 초래할 수 있다.

만약 데이터가 0와 999,999로 단 2개만 존재할 경우 데이터는 고작 2개지만
100만개의원소가 담길 수 있는 리스트가 필요하다..

2) 효율적일 때

계수 정렬은 동일한 값을 가지는 데이터가 여러개 등장할 때 효과적이다.

성적의 경우 100점 맞은 학생이 여러 명일 수 있기 때문에 계수 정렬이 효과적이다.

```
